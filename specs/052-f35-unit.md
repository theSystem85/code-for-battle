# Spec 052: F35 VTOL Stealth Strike Fighter

## Requirements
- Add `f35` as a buildable air unit with generated assets at `public/images/sidebar/f35_sidebar.webp` and `public/images/map/units/f35_map.webp`.
- Build prerequisites: player must have `(airstrip OR helipad) AND ammunitionFactory`.
- Cost must be 50% of F22 cost; max speed must be 50% slower than F22.
- F35 is a stealth air unit: invisible on enemy radar.
- F35 uses VTOL movement: it can take off and land like a helicopter, land on helipads, airstrip parking slots directly, and on any empty ground tile.
- When grounded on a helipad or airstrip, it auto-refills fuel and ammo.
- When grounded anywhere valid, it can be serviced by ammunition trucks, tanker trucks, and recovery tanks.
- F35 carries 6 JDAM bombs and can only attack ground targets.
- AGF support: when given multiple building targets, it should spend only the bombs required to destroy each target, then immediately continue to the next queued target with leftover bombs.
- Ground units may only target airborne F35 if they have anti-air capability.
- F35 may be targeted by F22 and Apache.
- Crashed F35 must use the same recoverable wreck flow/crash-sequence parity as F22.
- Use the `F35TakeOff`, `F35Landing`, `F35BombDrop`, and `F35BombDetonation` sounds for takeoff, landing, bomb release, and impact.
- Cheat unit spawning must support `f35`.

## Verification Targets
- Build/unlock path works from either helipad or airstrip plus ammunition factory.
- F35 can land on helipads, airstrips, and clear ground tiles.
- Airstrip/helipad landing refills fuel and ammo.
- Ammo/fuel/recovery support units can service a grounded F35.
- AGF bombing destroys multiple buildings without overspending bombs on earlier targets.
- Non-AA ground units ignore airborne F35; Apache/F22 can target it.

## 2026-03-08 Consistency fixes
- F35 bombs are release-gated by airborne combat state only (`airborne` + altitude + no landing/takeoff/taxi/park flags); grounded/taxi/landing/takeoff states cannot release bombs across AI/manual/remote/autoretaliation paths.
- Generic move intent no longer implies landing for F35. Landing transitions require explicit landing intent (`landAtStructure`, `explicitLand`, or `returnToBase`) with a valid landing target.
- Bomb delivery uses drop-style kinematics (inherited aircraft velocity + gravity/drag), no homing/thrust, and release occurs ahead of target based on speed-derived lead distance (1-3 tiles).
- F35 bombs render with no tracer/smoke trail; only bomb body + impact explosion/sound remain.
- Airstrip slot handling now supports reservation + occupancy separation so simultaneous F35 approaches cannot claim/overlap the same parking slot.
- F35 airborne flight audio now reuses F22 flight loop behavior while grounded/takeoff/landing sounds remain state-specific.
- F35 jet stream renderer now uses one centered tailpipe plume (single-engine layout) instead of dual plumes.

## 2026-03-08 Overflight and cadence adjustments
- F35 attack approach must use an overflight bombing run: when attacking, the jet flies through target center and only releases bombs once it is over (or crossing over) the target.
- F35 bomb release cadence is staggered: at most one bomb may release per attack update and release intervals must be at least 300ms.
- `public/images/map/units/f35_map.webp` must be regenerated from `output/imagegen/f35_map_source.webp` so the aircraft fills the 64x64 map sprite vertically (nose near top edge and tail near bottom edge).

## 2026-03-08 Verification additions
- E2E test validates that consecutive F35 bomb creation timestamps are separated by at least 300ms.
- E2E test validates that each bomb drop occurs while the F35 center is within over-target tolerance distance of the bomb target position.

## 2026-03-08 Enemy AI parity and targeting
- Enemy AI production must maintain a 1:1 ratio between planned/alive `f22Raptor` and `f35` units by producing an `f35` whenever F22 count exceeds F35 count and airstrip capacity allows it.
- Enemy-controlled `f35` strike targeting must prioritize defense-less ground buildings that are not covered by anti-air units/buildings.
- Coverage and eligibility checks must include both anti-air units (e.g., rocket tanks) and anti-air buildings (e.g., rocket turrets / anti-air defenses).
