# 051 — Street sprite-sheet routing (streets24 default)

1. The bundled sheet `images/map/sprite_sheets/streets24_q90_1024x1024.webp` must be listed in default static SSE sheet registries (`index.json`, map settings defaults, and SSE fallback list) so it is selectable without manual import. Retired street-sheet filenames must not be referenced by defaults.
2. Street-tile integrated rendering must resolve candidates using `street` plus directional tags (`top`, `bottom`, `left`, `right`) computed from orthogonal neighboring street tiles.
3. A directional tag is valid only when that matching orthogonal neighbor is a street tile (e.g. `top` only if the tile above is street).
4. Candidate street tiles containing any invalid directional tag must be excluded.
5. Candidate street tiles must contain exactly the same number and set of directional tags as the current orthogonal street-neighbor set; tiles with fewer or extra directional tags are not valid fallbacks for that street shape.
6. Street selection should prefer candidates also tagged with the active integrated biome tag (`grass`, `soil`, `snow`, `sand`) when available, with fallback to non-biome street candidates.
7. Street-hosted SOT rendering remains disabled in both CPU and WebGL overlay passes; terrain-hosted street SOT may render only for full-street cluster corners.
8. Street rendering must always paint a biome terrain underlay before the street overlay, so keyed/transparent street pixels reveal biome terrain rather than prior street/base artifacts.
9. Biome underlay for street tiles must use integrated biome-tagged land art when integrated sheets are enabled, and fall back to default grass/land terrain when integrated sheets are disabled.
10. Tagged street-sheet street rendering must remain active by default even when the `Custom sprite sheets` toggle is disabled.
11. In GPU base-layer mode, the CPU overlay pass must repaint street tiles using the same street underlay/overlay pipeline so tagged street-sheet rendering remains visible without requiring `Custom sprite sheets`.
12. Street-type SOT generation/rendering must be limited to land or water tiles bordered by full-eligible street tiles; water-type SOT must still render on street-hosted tiles so water/shore smoothing continues against the biome underlay beneath streets.
13. For street tiles adjacent to water corners, SOT texture content must use biome/land texture (selected integrated biome land when available, otherwise default land) rather than street texture so coastline smoothing visually matches the ground underlay.
14. Coastline smoothing beside streets must be oriented as land-on-water cut-in: street-hosted SOT remains disabled, and water tiles near street corners receive land SOT triangles so water no longer clips street overlays/units.
15. Land SOT drawn on water tiles near streets must sample land texture variation/biome from adjacent land/street source tiles (orientation-aware), not from the water tile coordinate, to avoid random gray/impassable wedges.
16. WebGL should continue rendering non-street land SOT overlays (e.g., inverse-island cases), but must skip land-SOT-on-water instances that are street-adjacent so street-coast smoothing comes from CPU biome-sampled wedges.
17. If a street tile has a diagonal street neighbor and the two orthogonal street neighbors that form that same corner (`top+left`, `top+right`, `bottom+left`, or `bottom+right`), street rendering must prefer candidates tagged `full` before regular directional candidates. This `full` match is dominant and intentionally avoids looped-looking repeated connector art inside wider street clusters.
18. Grass `top+right+bottom` T crossings must resolve to a grass-tagged exact directional candidate. In `streets24`, tile `4,4` is the no-left grass T candidate and tile `5,4` is the 4-way grass candidate; those metadata tags must remain distinct so fixing one shape cannot remove a required connection from the other.
19. Terrain-tile street SOT is allowed only when the two bordering street tiles for that corner are full-eligible street tiles. Non-full one-tile street corners and non-full T/corner pieces must not emit street SOT.
20. Street SOT fill art must be selected only from `street+full` tagged sprite-sheet candidates, preferring the active biome tag. It must not fall back to isolated/default street art or biome ground art, including when the SOT host tile is water.
